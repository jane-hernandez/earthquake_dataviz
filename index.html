Seismic activity Around the world.

Introduction

Our project is aimed at exploring the fascinating world of seismic activity around the world. We are all familiar with earthquakes, tsunamis and other ground shakingly humbling events that occur on our planet. But have you ever looked into the data about how often, or devistating these events can be? If not, we hope that you will find our project interesting.

Dataset

We decided to obtain our data from EveryEarthquake API (https://rapidapi.com/dbarkman/api/everyearthquake). Our initial data scraping provided 1,000 observations of earthquake type events but we will pull more data of the other types of seismic events for a more comprehensive analysis. The dataset obtained contains the following columns:

  • Index: An index assigned to each observation in the dataset.
  • Id: A combination of a 2-letter Seismic Network Code and a number assigned by the contributing seismic network.
  • Magnitude: The magnitude of the vibrations based on the Richter scale.
  • Type: The classification of the event based on its intensity. In our case, we only took earthquake-type rows.
  • Title: It indicates the event's name based on the origin's magnitude and location.
  • Date: It reports the date and time of the event. The timezone of reference for the time is not indicated.
  • Time: Time of origin of the earthquake using an UTC to avoid confusion caused by local time zones.
  • Updated: Time when the event was most recently updated.
  • Url: Link to USGS Event Page for event.
  • detailUrl: Link to the webpage with the details about the event.
  • Felt: This indicates the number of people who reported feeling the earthquake. This information may be absent from many rows because it is website-source specific.
  • Cdi: Stands for Community Decimal Intensity and is the result of assigning a single intensity to each community of people who reported an earthquake.
  • Mmi: It measures the damage level of the earthquake on the surface, and it’s a measure that goes from 0 to 12.
  • Alert: The alert level from the PAGER earthquake impact scale.
  • Status: Indicates whether the event has been reviewed by a human.
  • Tsunami: If the flag value is "1", the event will include a link to the NOAA Tsunami website for tsunami information.
  • Sig: A number describing how significant the event is. Determined on a number of factors, including: magnitude, maximum MMI, felt reports, and estimated impact.
  • Net: The ID of a data contributor. Identifies the network considered to be the preferred source of information for this event.
  • Code: An identifying code assigned by - and unique from - the corresponding source for the event.
  • Ids: A comma-separated list of event ids that are associated to an event.
  • Sources: Could be the website that provided the data.
  • Types: Type of seismic event.
  • Nst: Total number of seismic stations used to determine earthquake location.
  • Dmin: Horizontal distance from the epicenter to the nearest station (in degrees).
  • Rms: Root-mean-square (RMS) travel time residual, in sec, using all weights.
  • Gap: Largest azimuthal gap between azimuthally adjacent stations (in degrees).
  • Magtype: Method or algorithm used to calculate the preferred magnitude for the event.
  • Geometrytype: Geometry type of the distribution of the earthquake.
  • Depth: Depth of the event in kilometers, Hypocentral depth.
  • Latitude: Decimal degrees latitude. Negative values for southern latitudes. Range is [-90.0,90.0].
  • Longitude: Decimal degrees longitude. Negative values for western longitudes. [-180.0,180.0].
  • Place — textual description of named geographic region near to the event. This may be a city name, with additional distance measured in km.
  • distanceKM — distance measured in km of the earthquake.
  • placeOnly — Textual description of named geographic region near to the event. This may be a city name.
  • Location — network that originally authored the reported location of this event.
  • Continent — Continent where the event occurred.
  • Country — country where the event occurred (i.e. USA).
  • Subnational — Subnation where the event occurred (i.e. Alaska, Oregon…)
  • City — City where the event occurred.
  • Locality — Area or district where the event occurred.
  • Postcode — Postcode of the area affected by the earthquake.
  • What3words — Random 3 words associated with each event.
  • Timezone — Timezone of the area subject of the event.
  • locationDetails — further information on the earthquake location.

Task analysis

We aimed to split the project into three primary tasks. Namely:

  • Exploratory data analysis
  • Static graphic design
  • Dynamic graphic design

    • The first point looks into the data set to ensure that there is enough useful data and no repeated data for this project. This was to ensure that the data we display is of good quality and that the subject matter is objectively viable and useful. This process also included deriving new colums- namely the continent column from the the geolocations of the earthquakes.

    The and third points aimed to generate useful and detailed graphics that the user of our website can enjoy and interact with. Please do note that due the fact that the interactive apps are being hosted on a free server at the moment, the interactions may lag a little bit. The lag can be avoided if you decide to run the code form the git hub repository connected to this webpage.

    The final step of the project entailed creating some web pages to host our findings. This is the page you are corrently reading through. To display the interactive graphs on this webpage, the apps were uploaded and hosted on render and the pages were shown here using the .html 'iframe' functionality.
